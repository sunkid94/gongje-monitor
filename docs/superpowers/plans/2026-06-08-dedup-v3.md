# dedup v3 (AI 이벤트 라벨) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** enrich가 기사마다 정규화된 "이벤트 라벨"을 만들고 push_dedup가 그 라벨로 같은 사건을 묶어, 부고처럼 제목이 제각각인 사건이 푸시 1회만 가게 한다.

**Architecture:** enrich의 기존 Claude 호출(조합기사)에 `event_label` 필드를 추가(추가 호출 0)해 article에 부착한다. `push_dedup.filter_unpushed`는 article에 `event_label`이 있으면 그걸로 key/canon을 만들고, 없으면 기존 제목 기반으로 폴백한다. 기존 overlap·7일 창·방향가드·pushed.json은 그대로 재사용.

**Tech Stack:** Python 3.10+, anthropic SDK(기존), pytest 8.3.4.

설계 스펙: `docs/superpowers/specs/2026-06-08-dedup-v3-event-label-design.md`
대상: `enrich.py`(게이트 v2), `push_dedup.py`(dedup v2), `tests/test_enrich.py`, `tests/test_push_dedup.py`

## File Structure

| 파일 | 변경 |
|------|------|
| `enrich.py` | `_RELEVANCE_CRITERIA`/`_RELEVANCE_FIELD`에 event_label 추가; `enrich_article`가 event_label 파싱; `enrich_articles`가 article에 event_label 부착 |
| `push_dedup.py` | `from config import COMPANY_KEYWORDS, COMPANY_ALIASES`; `label_canon(label)`; `filter_unpushed`가 event_label 있으면 라벨로 key/canon |
| `tests/test_enrich.py` | event_label 생성·부착 테스트 |
| `tests/test_push_dedup.py` | label_canon + 라벨 기반 묶기/분리/폴백 테스트 |

**계약:** enrich article에 `event_label`(문자열) 추가(조합기사·AI가 줄 때). push_dedup는 그게 있으면 dedup 기준으로 사용, 없으면 제목 기반.

```python
def _normalize(text: str) -> str:
    return " " + " ".join(_PUNCT_RE.sub(" ", (text or "").lower()).split()) + " "


def label_canon(label: str) -> str:
    """이벤트 라벨에서 추적 조직(정식명/별칭)을 찾아 대표명 반환. 없으면 정규화 라벨 전체(폴백).

    AI 라벨이 대표조직명으로 시작하므로 보통 바로 매칭된다. 더 긴 이름 우선(짧은 별칭 오매칭 방지),
    공백 경계 매칭.
    """
    nlabel = _normalize(label)
    candidates = []
    for canon in COMPANY_KEYWORDS:
        for name in [canon] + COMPANY_ALIASES.get(canon, []):
            candidates.append((_normalize(name).strip(), canon))
    candidates.sort(key=lambda x: len(x[0]), reverse=True)
    for nname, canon in candidates:
        if nname and f" {nname} " in nlabel:
            return canon
    return nlabel.strip()
```

## Self-Review 결과

- **스펙 커버리지:** enrich event_label 프롬프트·파싱·부착(Task 1)↔스펙 "enrich"; label_canon + filter 라벨 우선·제목 폴백(Task 2)↔"push_dedup"; 부고 수렴·다른조직 분리·폴백(Task 2 테스트)↔스펙 "테스트" 3~6; 보수성(다른 조직 분리·라벨 없으면 폴백)↔"원칙". pushed.json 스키마 불변.
