# 공제조합 이슈 로그 웹사이트 설계

**작성일:** 2026-04-12  
**상태:** 승인됨

---

## 개요

기존 로컬 cron 기반 모니터링 시스템을 GitHub Actions로 이전하고, 수집된 기사를 공개 웹사이트(GitHub Pages)에 이슈 로그로 표시한다. 맥북이 꺼져 있어도 1시간마다 자동 수집되며, 누구든 공개 URL로 이슈 로그를 확인할 수 있다.

---

## 아키텍처

```
┌─────────────────────────────────────────────────┐
│              GitHub (무료)                        │
│                                                  │
│  ┌─────────────────┐      ┌───────────────────┐  │
│  │  GitHub Actions │      │   GitHub Pages    │  │
│  │  (매 1시간)      │      │   (웹사이트)       │  │
│  │                 │      │                   │  │
│  │ 1. 뉴스 수집     │─────▶│  articles.json    │  │
│  │ 2. 이메일 발송   │      │  index.html       │  │
│  │ 3. JSON 업데이트 │      │  (이슈 로그 페이지) │  │
│  └─────────────────┘      └───────────────────┘  │
│                                    │             │
└────────────────────────────────────┼─────────────┘
                                     ▼
                    https://[계정명].github.io/[저장소명]
```

**실행 흐름:**
1. GitHub Actions가 1시간마다 `main.py` 실행
2. 새 기사 발견 시 `articles.json`에 누적 저장 + 이메일 발송
3. 변경된 파일을 자동 커밋 → GitHub Pages 즉시 반영
4. 누구든 공개 URL로 이슈 로그 접근 가능

---

## 구성 요소

| 파일 | 역할 |
|------|------|
| `main.py` | `articles.json` 누적 저장 로직 추가 |
| `articles.json` | 수집된 기사 저장소 (자동 생성, 최대 500건) |
| `index.html` | 이슈 로그 웹페이지 (조합별 필터 포함) |
| `.github/workflows/monitor.yml` | GitHub Actions 1시간 스케줄 |

---

## articles.json 구조

```json
[
  {
    "keyword": "기계설비건설공제조합",
    "title": "기계설비건설공제조합, 신규 사업 발표",
    "link": "https://...",
    "description": "내용 요약...",
    "collected_at": "2026-04-12T13:45:00"
  }
]
```

- 최신 기사가 배열 앞에 위치 (날짜 역순)
- 최대 500건 유지 (초과 시 오래된 것부터 삭제)

## GitHub Actions 설정

```yaml
# .github/workflows/monitor.yml
schedule:
  - cron: '0 * * * *'  # 매 1시간 정각
```

**GitHub Secrets에 저장할 값:**
- `GMAIL_ADDRESS`
- `GMAIL_APP_PASSWORD`
- `RECIPIENTS`

민감 정보는 코드에 노출되지 않고 Secrets에서 환경변수로 주입된다.

---

## 기존 시스템 변경사항

- 맥북 cron 제거 (더 이상 불필요)
- `seen.json` → GitHub Actions 실행 환경에서 관리 (Actions가 매 실행마다 저장소에서 pull)
- `config.env` → GitHub Secrets로 대체

---

## 제약사항

- GitHub 저장소는 public으로 설정해야 GitHub Pages 무료 사용 가능
- `articles.json`은 저장소에 커밋되므로 공개됨 (뉴스 기사 제목/링크이므로 문제 없음)
- GitHub Actions 무료 플랜: 월 2,000분 (1시간마다 실행 시 월 약 720분 사용, 충분)
